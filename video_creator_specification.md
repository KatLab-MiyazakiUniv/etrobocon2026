# 高速画像動画化ツール (Python + FFmpeg) 技術仕様書

本仕様書は、実機（またはシミュレータ）のカメラから取得し一時保存された連続JPEG画像から、画像処理（ROI枠・文字の描画）を施しつつ、FFmpegパイプラインを利用して高速にMP4動画を作成するコマンドラインツールの仕様を定めたものである。

---

## 1. 背景と目的

昨年度のシステム（C++による同期保存および `cv::VideoWriter` による動画作成）では、以下の2つのフェーズで重大な処理遅延が発生していた。
1.  **走行中の遅延**: カメラ画像のディスク保存（`cv::imwrite`）が同期的かつメインスレッドで行われていたため、制御ループをブロックし、走行性能に悪影響を与えていた。
2.  **走行後の動画化遅延**: 大量のJPEG画像のデコードと、CPUによるシングルスレッドでのH.264エンコードをすべて直列に行っていたため、動画作成に数分以上の時間を要していた。

### 解決アプローチ
*   **走行中**: 画像を保存するディレクトリをメモリ上（RAMディスクや一時フォルダ）に配置する、または非同期化する。
*   **走行後（本ツール）**: Pythonで画像の読み込み・描画（ROI枠等の合成）を行い、圧縮処理を行わずに生の画像バイナリデータをそのままパイプ（標準入力）経由でバックグラウンドの **FFmpegプロセス** に流し込む。これにより、重いエンコード処理がFFmpeg側にオフロードされ、マルチスレッドやGPUアクセラレーションの恩恵を最大限に受けることができ、動画化処理を劇的に高速化する。

## 3. 機能要件

### 3.1. 入力処理
*   **画像ファイルの収集**: 指定された入力ディレクトリから、ファイル名が `roi_` で始まり、拡張子が `.JPEG`（または `.jpg`, `.png`）であるファイルを走査する。
*   **タイムスタンプソート**: ファイル名の末尾に含まれるタイムスタンプ（ナノ秒またはミリ秒等のエポックタイム数値）を抽出し、撮影された時系列順に正しくソートする。
    *   *ファイル名規則例*: `roi_x{x}_y{y}_w{w}_h{h}_{timestamp}.JPEG`

### 3.2. 画像処理（描画・合成）
*   **リサイズ（縮小）**: 動画の容量削減と処理高速化のため、元の解像度から指定された比率（デフォルト：`0.5`倍）に縮小（リサイズ）する。
*   **ROI（注目領域）矩形の描画**: ファイル名からROI座標（`x`, `y`, `width`, `height`）をパースし、リサイズ後のスケールに合わせて座標を変換した上で、画像上に赤い枠（矩形、太さ2px）を描画する。
*   **テキスト情報の重畳**: パースしたROIの生座標値（例: `ROI: x=100, y=120, w=200, h=150`）を、画像の左上等に視認しやすい色（赤または白）のテキストで描画する。

### 3.3. 動画出力（FFmpegパイプライン）
*   **FFmpegサブプロセスの起動**: Pythonの `subprocess` モジュールを使用し、FFmpegをサブプロセスとして起動する。
*   **パイプライン接続**: FFmpegの入力仕様を「rawvideo（生画像データ）」「BGR24形式」「標準入力（stdin）受け取り」に設定する。
*   **エンコード処理**: 指定されたフレームレート（FPS）およびコーデック（H.264等）でMP4形式の動画を出力する。
    *   **GPUサポート**: 引数オプションでGPUアクセラレーション（例: NVIDIAの `h264_nvenc`）を有効化できるようにする。

### 3.4. コマンドライン・インターフェース (CLI)
ツールは以下のコマンドライン引数を受け付けるようにする。

| 引数名 | 短縮名 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- |
| `--input-dir` | `-i` | `datafiles/line_trace` | 入力JPEG画像が格納されているディレクトリパス |
| `--output` | `-o` | `line_trace.mp4` | 出力するMP4動画のファイルパス |
| `--fps` | `-r` | `15` | 出力動画のフレームレート（フレーム/秒） |
| `--scale` | `-s` | `0.5` | 画像の縮小比率 (0.1 〜 1.0) |
| `--gpu` | `-g` | `False` (フラグ) | NVIDIA GPUエンコーダー (`h264_nvenc`) を使用するかどうか |
| `--clean` | `-c` | `False` (フラグ) | 動画作成完了後に、入力画像ディレクトリ内のファイルをすべて削除するかどうか |

---

## 4. 非機能要件・環境要件

*   **動作環境**: Linux (Ubuntu/WSL2) および Windows。
*   **必須ソフトウェア**:
    *   Python 3.8 以上
    *   FFmpeg (システム環境変数 PATH に登録されていること、または実行可能ファイルが利用可能であること)
*   **Python依存ライブラリ**:
    *   `opencv-python` (画像デコード、描画用)
*   **パフォーマンス目標**: 1000枚の画像（解像度 800x600）を、30秒以内にROI枠付きMP4動画へ結合完了すること。

# 仕様実現用 AIプロンプト (Prompt for Coding Agent)

以下は、上記の仕様書に従ってPythonプログラムを別のAIに生成させるためのプロンプトである。

```text
あなたは高度な技術を持つPythonエンジニアです。
提供された仕様書に基づき、画像ファイル群をFFmpegのパイプ機能を用いて高速に動画化するスクリプトを作成してください。

### 【作成するファイル名】
`create_line_trace_video.py`

### 【満たすべき実装要件】

1.  **依存ライブラリと外部プロセスの起動**:
    *   画像のデコードと描画、リサイズには `opencv-python` (cv2) を使用してください。
    *   `subprocess.Popen` を使用して `ffmpeg` コマンドを非同期プロセスとして起動し、その標準入力 (`stdin`) に生の画像データを書き込む構造にしてください。
    *   FFmpegプロセスの引数は以下を基本としてください。
        `ffmpeg -y -f rawvideo -vcodec rawvideo -s {width}x{height} -pix_fmt bgr24 -r {fps} -i - -c:v libx264 -pix_fmt yuv420p {output_path}`
    *   `--gpu` フラグが指定された場合は、`-c:v libx264` を `-c:v h264_nvenc` に切り替えて起動できるようにしてください。

2.  **画像ファイル走査とソート**:
    *   指定された `--input-dir` 内の、ファイル名が `roi_` から始まり拡張子が `.JPEG` または `.jpg` であるファイルを検索してください。
    *   ファイル名は `roi_x{x}_y{y}_w{w}_h{h}_{timestamp}.JPEG` という形式です。
    *   ファイル名の末尾にある `timestamp`（数値部分）を正規表現等で抽出し、数値としてソート（昇順）したリストを作成してください。これにより撮影順にソートします。

3.  **描画処理**:
    *   各画像を読み込み（`cv2.imread`）、指定された `--scale` 比率で縮小（`cv2.resize`）してください。
    *   ファイル名から `x`, `y`, `w`, `h` の数値をパースし、スケール比率を掛けた座標に変換した上で、縮小後の画像に赤い長方形（太さ2）を描画してください。
    *   さらに、縮小後の画像上に `ROI: x={x}, y={y}, w={w}, h={h}` というテキスト（赤色、フォントは `cv2.FONT_HERSHEY_SIMPLEX`、サイズや太さは適切に調整）を描画してください。

4.  **パイプ処理のライフサイクル管理**:
    *   すべての画像を処理して `proc.stdin.write(resized.tobytes())` で書き込んだ後、必ず `proc.stdin.close()` を呼び出し、`proc.wait()` でFFmpegプロセスが正常終了するのを待機してください。

5.  **例外処理とディレクトリのクリーンアップ**:
    *   画像の読み込みに失敗した場合（破損ファイルなど）は、エラーを出力してそのフレームをスキップし、処理を続行してください。
    *   `--clean` フラグが指定されている場合、動画の作成が完全に成功した後に、入力ディレクトリ内の画像ファイルを削除してください（安全のため、ディレクトリ自体の削除ではなく、走査した対象ファイルを削除する設計にしてください）。

6.  **CLI引数設計**:
    *   Pythonの `argparse` モジュールを使用して、以下の引数を実装してください。
        *   `-i`, `--input-dir` (default: 'datafiles/line_trace')
        *   `-o`, `--output` (default: 'line_trace.mp4')
        *   `-r`, `--fps` (type: int, default: 15)
        *   `-s`, `--scale` (type: float, default: 0.5)
        *   `-g`, `--gpu` (action: 'store_true', default: False)
        *   `-c`, `--clean` (action: 'store_true', default: False)

### 【コードの品質基準】
*   コードには適切な日本語のコメントや docstring を記述し、可読性を高めてください。
*   スクリプトが単体で実行可能なように `if __name__ == '__main__':` ブロックを設けてください。
*   処理の進捗が分かるように、進捗バーを表示するか、または「N / Total 枚処理完了」といったログを標準エラー出力/標準出力に定期的（例: 50枚ごと）に表示する機能を実装してください。
```
